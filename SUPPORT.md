# Support

Use GitHub issues for public-safe questions and reproducibility reports.

## Good Support Requests

- `doctor` or `smoke` fails on a clean checkout;
- a documented command is confusing or stale;
- a manifest row, license note, or report index looks inconsistent;
- a claim family seems relevant but has no registered template;
- a public example output is hard to interpret.

## Please Do Not Post

- confidential paper text or private review notes;
- author identities from double-blind review;
- raw datasets or files with unclear redistribution rights;
- API keys, credentials, private repository links, or local absolute paths;
- requests for acceptance, rejection, legal, ethics, or deployment decisions.

## Useful Details To Include

- repository commit or release tag;
- operating system and Python version;
- exact command you ran;
- public-safe output excerpt;
- whether the failure occurs in a fresh checkout.

For vulnerability, privacy, or accidental-disclosure concerns, see
`SECURITY.md`.
